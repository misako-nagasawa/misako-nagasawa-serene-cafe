# Handoff: Sérène Café — ブランドサイト / ランディングページ

## 概要 (Overview)
「Sérène（セレーヌ）」は新規オープンしたカフェのための1ページ完結型のブランドサイト（ランディングページ）です。来店促進を目的に、お店の世界観・コンセプト・メニュー・アクセス・予約導線をスクロール1本で伝えます。

- **デザインの方向性**: ナチュラル × 北欧スカンジナビア。木・植物・ベージュの温かみ。スタイリッシュになりすぎない、落ち着いた佇まい。
- **配色**: アースカラー（ブラウン × テラコッタ × くすみベージュ、差し色に植物のセージグリーン）。
- **店名の由来**: 「Sérène」はフランス語で「穏やか」。タグライン = *un moment de douceur.*

## このバンドルのファイルについて (About the Design Files)
このフォルダの `index.html` は **HTMLで作られた完成デザイン（自己完結型の静的サイト）** です。

このサイトは外部依存が **Google Fonts (CDN) のみ** で、ビルド工程・サーバー・データベースを一切必要としません。したがって2つの進め方が可能です:

1. **そのままデプロイする（推奨・最短）** — `index.html` を静的ホスティングに上げるだけで公開できます。下記「デプロイ手順」を参照。
2. **既存コードベースに作り直す** — 既にReact/Next.js/Astro等の環境がある場合は、本READMEを仕様書として、その環境のパターンに沿って再実装してください。

> このサイトには本番フレームワークが存在しないため、**「そのままデプロイ」が最も適切**です。フレームワーク化は要件があるときのみ行ってください。

## デプロイ手順 (Deployment) — Claude Code 向け

このサイトは静的1ファイル（`index.html`）です。以下のいずれかで公開できます。**Vercel か Netlify が最も手軽です。**

### 前提
- 画像はすべてプレースホルダーです。**公開前にお店の実写真へ差し替えてください**（後述「写真の差し替え」）。
- 店舗情報（住所・電話・営業時間・SNSリンク）は仮値です。**実データに更新してください**（後述）。

### A. Vercel CLI でデプロイ
```bash
cd design_handoff_serene_cafe
npx vercel --prod
# 初回はログイン → プロジェクト名を設定 → 出力されたURLが本番URL
```
`index.html` がルートにあるため、追加設定なしで配信されます。

### B. Netlify CLI でデプロイ
```bash
cd design_handoff_serene_cafe
npx netlify deploy --prod --dir .
# 初回はログイン → サイト作成 → デプロイ
```

### C. GitHub Pages
```bash
cd design_handoff_serene_cafe
git init && git add . && git commit -m "Sérène Café site"
# GitHubに新規リポジトリ作成 → push
git branch -M main
git remote add origin <your-repo-url>
git push -u origin main
# リポジトリ Settings → Pages → Source: main / (root) を選択
```

### 独自ドメイン
- VercelもNetlifyも管理画面の「Domains」から独自ドメイン（例: `serene-cafe.jp`）を接続できます。DNSのCNAME/Aレコード設定はホスティング側の案内に従ってください。

- [ ] スマホ表示でハンバーガーメニューが開閉する

## 画面構成 (Screens / Sections)
単一ページの縦スクロール。固定ヘッダー + 7セクション。

### 1. ヘッダー / ナビ (fixed)
- **構成**: 左にロゴ `Sérène.`、右にナビリンク（Concept / Menu / Gallery / Access）+ 丸ピル型CTA「ご予約」。
- **挙動**: スクロール40px超で `.scrolled` クラスが付き、背景が半透明クリーム + ブラー + 下罫線に変化、パディング縮小。最上部（ヒーロー上）では文字が白、スクロール後はブラウン。
- **モバイル(≤880px)**: ナビは右からスライドインするドロワー + 暗幕(backdrop)。ハンバーガーは×に変形。

### 2. ヒーロー (Hero)
- **レイアウト**: `min-height:100svh`、コンテンツは下寄せ(`flex-end`)。背景は全面プレースホルダー画像 + 暗いブラウンのグラデーションオーバーレイ。
- **要素**: アイブロウ `CAFÉ & ROASTERY — depuis 2026` / 特大ロゴ見出し `Sérène`（"r" のみテラコッタのイタリック）/ 縦書きの "Scroll" キュー（アニメーションするバー付き）。
- **コピー**: 「木のぬくもりと、植物の緑。/ ゆっくりと、こころほどける時間を。/ — un moment de douceur.」

### 3. コンセプト (Concept)
- **レイアウト**: 2カラム（左テキスト 1.05fr / 右ビジュアル 0.95fr）。右はメイン画像(4:5) + 重ねた正方形のフロート画像 + 「douceur」のフローティングノート。
- **コピー**: 店名の由来と空間の説明（無垢材・観葉植物・北欧の住まいのような雰囲気）。

### 4. バリュー (Values) — 背景 `--bg-2`
- 3カラム。各カラム = 番号(01–03 イタリック) + 見出し + 説明。
- 内容: ①自家焙煎の一杯 ②植物と木の空間 ③毎朝の焼き菓子。

### 5. メニュー (Menu)
- **レイアウト**: 見出し行（左タイトル / 右に注記）+ 2カラムのメニューリスト。
- **カテゴリ**: Coffee / Tea & Others / Sweets / Light Meal。
- **行スタイル**: 品名（和文 + 下に英語小文字キャプション） … 点線リーダー … 価格。価格はすべて税込・**仮価格**。

### 6. ギャラリー (Gallery) — 背景 `--bg-2`
- 4列のメイソンリー風グリッド（`grid-auto-rows:200px`、一部 `span 2`）。ホバーで微拡大 + 明度UP。プレースホルダー8枚。

### 7. アクセス・営業時間 (Access)
- 2カラム。左に定義リスト(Open / Closed / Address / Tel) + 「テラス席・Wi-Fi・電源あり」ピル。右に正方形のMAPプレースホルダー（**Google Maps埋め込みに差し替え**）。
- **仮データ**: 9:00–19:00 / 定休 水曜 / 東京都〇〇区〇〇 1-2-3 / 03-0000-0000。

### 8. 予約・お問い合わせ (Reserve) — 背景 `--brown`（濃いブラウン地に明色テキスト）
- 2カラム。左に見出し + 説明。右にフォーム（お名前* / 電話番号* / ご希望日* / 人数 / ご要望）。
- **送信挙動**: クライアント側バリデーション。必須未入力は下罫線が赤 + エラーテキスト表示。OKならフォームを隠して「ありがとうございます…」の完了メッセージに切替。`date` は当日以降のみ選択可(`min`)。
- ※現状はフロントのみ。**実運用では送信先（メール/予約システム/Googleフォーム等）への接続が必要**。

### フッター (Footer) — 背景 濃ブラウン
- 左にロゴ + タグライン、右に3カラム（Menu / Hours / Follow=Instagram・X・LINE）。下部にコピーライト。SNSリンクは現状ダミー(`#`)。

## インタラクション & 挙動 (Interactions & Behavior)
- **固定ナビのスクロール状態変化**: `scrollY > 40` で背景/文字色/余白がトランジション（0.5s `cubic-bezier(.22,.61,.36,1)`）。
- **モバイルドロワー**: ハンバーガーで開閉、リンク/暗幕タップで閉じる、開いている間は `body` スクロールロック。
- **スクロールリビール**: `.reveal` 要素が `opacity:0 / translateY(28px)` から `.in` で表示（1s, `.d1/.d2/.d3` で遅延）。IntersectionObserver + ビューポート判定のフォールバックを併用。`prefers-reduced-motion:reduce` では即時表示。
- **ギャラリーホバー**: `scale(1.015)` + 明度。
- **フォームバリデーション**: 必須 = name, tel, date。`input` 時にエラー解除。
- **スクロールキュー**: 縦バー内のインジケータが2.4sループ。

**タイポグラフィ（Google Fonts）**
- 見出し/ロゴ: **Cormorant Garamond**（400/500, italic）— ラテン。和文フォールバック Shippori Mincho。
- 和文見出し/引用: **Shippori Mincho**（400/500）。
- 本文（和文）: **Zen Kaku Gothic New**（300/400/500）。
- ラベル/英字キャプション/ボタン: **Jost**（300/400/500、`letter-spacing` 広め + `uppercase`）。

**スケール / その他**
- コンテンツ最大幅: `--maxw: 1240px`、左右パディング `clamp(20px,5vw,64px)`。
- セクション縦パディング: `clamp(80px,12vh,150px)`。
- 角丸: 主に `2–3px`（控えめ）、ピル/CTAは `100px`。
- イージング: `--ease: cubic-bezier(.22,.61,.36,1)`。
- ブレークポイント: `880px`（レイアウト2→1カラム化・モバイルナビ）、`520px`（微調整）。
- 行間 `line-height:1.85`、字間 `letter-spacing:.02em`。

## 状態管理 (State Management)
フレームワーク無しのバニラJS。管理する状態は最小限:
- ナビのスクロール状態（`.scrolled` クラスのトグル）
- モバイルメニュー開閉（`.open` クラス + body スクロールロック）
- リビール済み要素の集合（IntersectionObserver / 配列から除去）
- フォームのバリデーション結果と送信完了フラグ（`.invalid` / 完了メッセージ表示）

Reactで再実装する場合は `navScrolled`, `menuOpen`, `formErrors`, `submitted` などの state に対応。

## 写真・店舗情報の差し替え (Assets & Content to Replace)
**画像**: 現状はCSSで描いたストライプ柄のプレースホルダー（`.ph` クラス、`data-label` に用途を明記）。以下を実写真に差し替え:
- ヒーロー全面ビジュアル（推奨 1920×1280 以上、横長）
- コンセプト: 店内インテリア(縦 4:5) + コーヒー(正方形)
- ギャラリー8枚（店内全景/カウンター/コーヒー/焼き菓子/窓辺の席/植物/ラテアート/外観）
- アクセス: Google Maps の `<iframe>` 埋め込みに置換

差し替え方法: `<div class="ph" data-label="...">` を `<img src="..." alt="...">`（または `background-image`）に置き換え。`aspect-ratio` / `object-fit:cover` を維持すると崩れません。

**テキスト/情報**: 店名・キャッチコピー・メニュー名と価格・営業時間・住所・電話・SNSリンク(`href="#"`) はすべて仮。実データに更新してください。
